컴퓨터 구조 기초 정리
#

오늘 대화를 바탕으로 정리한 학습 노트
“암기가 아닌 이해” — 왜 탄생했는지부터 시작하는 방식으로 학습

이 글을 쓴 이유
#

컴퓨터는 0,1밖에 모르는 데, 어떻게 복잡한 프로그램이 작동하는 것인지, 궁금했고 그 답을 찾아가는 과정을 정리했다.

목차
#

  1. 컴퓨터의 어원
  2. 왜 기계 컴퓨터를 만들게 됐나
  3. 불 대수 — 0과 1의 탄생
  4. 논리 게이트 — 전기로 논리를 구현하다
  5. 진공관 → 트랜지스터 → CPU
  6. 메모리 계층 구조
  7. 전체 흐름 요약

1. 컴퓨터의 어원
#

어원
#

  • Computer = Compute(계산하다) + er(하는 사람)
  • 1600년대부터 계산을 전문으로 하는 사람을 “computer"라고 불렀음
  • 천문학자, 항해사, 수학자들이 복잡한 수치 계산을 담당하는 직업이었음

핵심
#

컴퓨터는 원래 기계가 아니라 사람이었다

2. 왜 기계 컴퓨터를 만들게 됐나
#

계기 1: 인간 계산의 한계 (1820년대)
#

인간 computer에게 계산을 맡기면 두 가지 문제가 있었다.

① 느리다
② 반드시 실수를 한다

찰스 배비지(Charles Babbage) 는 항해용 수학표(로그표, 삼각함수표)의 오류가 선박 침몰로 이어지는 것을 보고 이렇게 생각했다.

“기계가 계산하면 실수가 없지 않을까?”

→ Difference Engine(차분기관) 설계 (1820년대) — 최초의 기계식 계산기 개념

계기 2: 전쟁의 긴박함 (1940년대)
#

2차 세계대전에서 두 가지 문제가 등장했다.

문제내용한계
독일군 암호(에니그마) 해독매일 수백만 가지 경우의 수사람이 계산하면 전쟁이 끝나도 못 풀 속도
포탄 탄도표 계산발사 각도별 계산표 필요사람이 하면 하나에 40시간 소요

결과물:

  • 영국: Colossus(1943) — 암호 해독용
  • 미국: ENIAC(1945) — 탄도표 계산용, 사람 40시간 → 30초

※ 영화 [이미테이션 게임]에 나오는 튜링의 기계는 Bombe(봄베) Colossus와는 별개의 암호 해독기

배비지 vs 전쟁의 차이
#

배비지: 인간의 실수 → 정확한 기계가 필요
전쟁:   시간 부족  → 빠른 기계가 필요

3. 불 대수 — 0과 1의 탄생
#

조지 불(George Boole)이란?
#

  • 1800년대 영국의 수학자이자 논리학자
  • 불 대수(Boolean Algebra) 창안
  • 프로그래밍의 boolean이 이 사람 이름에서 온 것

불 대수의 핵심 아이디어
#

“모든 논리는 참(True) / 거짓(False) 두 가지로 표현할 수 있다”

기본 논리 연산
#

연산의미예시
AND둘 다 참이어야 참비가 오고 AND 우산이 있으면 → 안 젖음
OR하나만 참이어도 참버스 OR 지하철 → 하나만 있어도 이동 가능
NOT반대로 뒤집음참 → 거짓, 거짓 → 참

0과 1이 왜 중요한가
#

전기 = 켜지거나 꺼지거나, 딱 두 가지 상태만 존재
전기 흐름  = 1 (참)
전기 없음  = 0 (거짓)

→ 0.5 같은 중간 상태가 없음
→ 전기로 표현할 수 있는 가장 단순한 상태
→ 동시에 모든 논리와 계산의 기본 단위

4. 논리 게이트 — 전기로 논리를 구현하다
#

논리 게이트란?
#

불 대수의 AND, OR, NOT을 전기 회로로 구현한 것

트랜지스터와 논리 게이트의 관계
#

트랜지스터 1개 = 전기 스위치 1개 = 0 또는 1 표현
논리 게이트 1개 = 트랜지스터 여러 개의 조합

트랜지스터 → 논리 게이트 → 연산 회로 → CPU
  (재료)        (부품)        (기능)      (완성품)

각 게이트의 트랜지스터 수
#

게이트스위치 수동작
AND2개둘 다 켜져야 1 출력
OR2개하나만 켜져도 1 출력
NOT1개입력의 반대를 출력

논리 게이트 → 덧셈 회로
#

컴퓨터는 이진수로 계산한다.

사람:   1 + 1 = 2
컴퓨터: 1 + 1 = 10 (이진수)
         → 현재 자리 = 0
         → 올림수   = 1

이걸 게이트로 구현하면:

현재 자리 결과 = A XOR B
올림수        = A AND B
→ 이것이 반가산기(Half Adder)

흐름:

논리 게이트 조합 → 덧셈 회로
덧셈 회로 여러 개 → 사칙연산
사칙연산 + 기억장치 → 계산기 = 컴퓨터

5. 진공관 → 트랜지스터 → CPU
#

진공관(Vacuum Tube)
#

  • 유리로 된 전구 모양의 부품
  • 전기 스위치 역할 (0과 1 표현)
  • 문제점:
크기: 손가락만큼 큼
열:  엄청난 발열
수명: 자주 고장 (ENIAC은 매일 몇 개씩 터짐)

트랜지스터(Transistor, 1947)
#

  • 진공관과 같은 역할 (전기 스위치 = 0과 1)
  • 하지만 완전히 다른 스펙
진공관 → 트랜지스터
크기:  손가락  →  손톱보다 작음
발열:  엄청남  →  적음
수명:  자주 고장 →  안정적

CPU의 탄생과 발전
#

1945년 ENIAC
→ 방 한 칸 크기 (167㎡), 무게 27톤
→ 진공관 18,000개

1971년 인텔 4004 (최초의 CPU)
→ 손톱만한 칩 하나
→ 트랜지스터 2,300개

2024년 애플 M4
→ 손톱만한 크기 동일
→ 트랜지스터 280억 개

같은 손톱 크기에 트랜지스터가 1,200만 배 증가

CPU란?
#

트랜지스터 수십억 개가 집적된 계산 처리 장치 논리 게이트의 조합으로 빠르게 계산하는 부품

CPU가 빠른 이유 3가지
#

① 트랜지스터 수십억 개 → 동시에 엄청난 계산 가능
② 클럭 속도가 높음     → 1초에 수십억 번 연산
③ 캐시 메모리          → 자주 쓰는 데이터를 바로 옆에 저장

클럭 속도(Clock Speed)란?
#

  • Clock = 시계
  • CPU 안에 아주 빠르게 똑딱거리는 시계가 있음
  • 한 번 똑딱할 때마다 연산 한 번 수행
클럭 속도 1Hz  = 1초에 1번 연산
클럭 속도 1GHz = 1초에 10억 번 연산
클럭 속도 3GHz = 1초에 30억 번 연산

컴퓨터 역사 흐름
#

조지 불 (1850s)
"논리를 0과 1로 표현 가능"
        ↓
논리 게이트 개념 (1930s)
"0과 1을 전기로 구현 가능"
        ↓
진공관 컴퓨터 (1940s)
"게이트 수만 개 → ENIAC 탄생"
        ↓
트랜지스터 (1947)
"진공관보다 작고 안정적"
        ↓
집적회로/CPU (1971~)
"트랜지스터 수십억 개를 손톱 크기에"
        ↓
PC (1977~) → 스마트폰 (2007~) → 지금

6. 메모리 계층 구조
#

메모리(Memory)란?
#

  • Memory = 기억하다(memorize)에서 온 말
  • 데이터를 저장하고 필요할 때 꺼낼 수 있는 모든 장치
  • 현업에서 “메모리 부족” = 보통 RAM이 부족하다는 의미

저장장치 비교
#

HDD (Hard Disk Drive)
#

물리적으로 원판(플래터)을 돌려서 데이터를 읽음
→ 기계적 한계 존재
→ 충격에 약함
→ 느림

SSD (Solid State Drive)
#

  • Solid State = 고체 상태 = 움직이는 부품이 없다
  • 플래시 메모리 방식으로 데이터 저장
플래시 메모리 = 전자를 물리적으로 가두는 방식
전자 있음 = 1
전자 없음 = 0
전기 없어도 데이터 유지됨

트랜지스터 vs 플래시 메모리 차이
#

구분트랜지스터플래시 메모리
방식전기 신호로 켜고 끔전자를 물리적으로 가둠
목적계산, 논리 처리데이터 영구 보관
전기 끊기면상태 사라짐데이터 유지
비유전등 스위치자물쇠

RAM (Random Access Memory)
#

  • 프로그램 실행 중 필요한 데이터를 임시 저장
  • CPU가 필요할 때 빠르게 꺼내 쓰는 작업대
  • 전기 끊기면 데이터 사라짐
DRAM (Dynamic RAM) — RAM에 사용
→ 트랜지스터 1개 + 커패시터로 1비트 저장
→ 커패시터에서 전하가 새어나감
→ 주기적으로 재충전(Refresh) 필요
→ Refresh 때문에 느림
→ 구조 단순 → 대량생산 용이 → 저렴

커패시터(Capacitor)란?

  • Capacity(용량)에서 온 말
  • 전기를 잠깐 저장할 수 있는 부품
  • 충전 = 1, 방전 = 0
  • 시간이 지나면 전하가 새어나가는 문제 있음

캐시 메모리 (Cache Memory)
#

  • Cache = 프랑스어로 “숨겨둔 것”
  • CPU 안에 붙어있는 초고속 소용량 메모리
SRAM (Static RAM) — 캐시에 사용
→ 트랜지스터 6개로 1비트 저장
→ 서로 맞물린 구조로 전기만 있으면 상태 유지
→ Refresh 불필요
→ 매우 빠름
→ 구조 복잡 → 비쌈

SRAM vs DRAM 비교

구분SRAMDRAM
속도매우 빠름보통
가격비쌈저렴
구조트랜지스터 6개/비트트랜지스터 1개 + 커패시터/비트
Refresh불필요필요
용도캐시 메모리RAM

메모리 계층 구조 전체
#

빠름 ↑   비쌈 ↑   용량 ↓
─────────────────────────────────
CPU 레지스터   CPU 안, 극소량 (B 단위)
캐시  SRAM    CPU 안, KB~MB
RAM   DRAM    메인보드, GB
SSD   플래시   저장장치, TB
HDD   자기     저장장치, TB
─────────────────────────────────
느림 ↓   쌈 ↓    용량 ↑

속도 차이를 거리로 비유하면
#

CPU 캐시에서 읽기  = 내 손에서 집기 (1초)
RAM에서 읽기      = 옆방에서 가져오기 (수십 초)
SSD에서 읽기      = 편의점 다녀오기 (수백 초)
HDD에서 읽기      = 마트 다녀오기 (수천 초)

버스(Bus)란?
#

  • 라틴어 Omnibus(모든 사람을 위한) 에서 온 말
  • 여러 부품 사이에 데이터를 실어 나르는 통로
CPU ←──버스──→ RAM ←──버스──→ SSD

프로그램 실행 흐름
#

SSD (냉장고)
→ 버스 타고
→ RAM (요리대)에 올라옴  ← 이게 로딩 시간
→ CPU (요리사)가 RAM에서 꺼내 계산
→ 자주 쓰는 건 캐시(앞주머니)에 보관

로딩 시간 = SSD에서 RAM으로 데이터가 이동하는 시간

가상 메모리(Virtual Memory)
#

크롬 탭을 많이 열면:

탭 많아짐
→ 각 탭의 데이터가 RAM에 저장
→ RAM 용량 초과
→ 운영체제가 개입
   ① 안 쓰는 데이터를 RAM에서 제거
   ② 또는 SSD의 일부 공간을 RAM처럼 사용 → 가상 메모리
→ SSD는 태생적으로 RAM보다 느림
→ 컴퓨터가 느려짐 (렉 발생)

컴퓨터 입장에서는 데이터를 기다렸다가(페이지 폴트) 받으면 재개하는 것 사용자 눈에는 버벅거림으로 보임

이를 전문용어로 페이지 폴트(Page Fault) 라고 함

7. 전체 흐름 요약
#

1850s  조지 불      → 불 대수: 논리를 0과 1로 표현
1820s  찰스 배비지  → 기계식 계산기 필요성 인식
1940s  2차 세계대전 → ENIAC: 최초 전자식 컴퓨터 (진공관 18,000개)
1947   트랜지스터   → 진공관 대체, 소형화 시작
1971   인텔 4004    → 최초 CPU, 트랜지스터 2,300개
2024   애플 M4      → 트랜지스터 280억 개 (1,200만 배 증가)

핵심 개념 한 줄 정리
#

개념한 줄 정의
트랜지스터전기 스위치. 0과 1을 표현하는 가장 작은 단위
논리 게이트트랜지스터 조합으로 만든 AND/OR/NOT 회로
CPU트랜지스터 수십억 개가 집적된 계산 처리 장치
클럭 속도CPU가 1초에 몇 번 연산하는지 (GHz)
캐시(SRAM)CPU 안의 초고속 소용량 메모리
RAM(DRAM)프로그램 실행 중 데이터를 임시 저장하는 작업대
SSD플래시 메모리 방식의 빠른 저장장치
HDD원판을 물리적으로 돌리는 저장장치
버스부품 간 데이터를 실어 나르는 통로
가상 메모리RAM 부족 시 SSD 일부를 RAM처럼 사용하는 것
페이지 폴트가상 메모리에서 데이터를 기다리는 상황
